# Mandatory Disclaimers for Clinical AI

Every response generated by a healthcare AI system must include a disclaimer
that clarifies the system's limitations. This is not optional -- it is required
by regulatory frameworks including CFM Resolution 2.338/2023 (Brazil), FDA
guidance on Clinical Decision Support (US), and MDR Article 63 (EU).

## Regulatory Requirements

### CFM (Conselho Federal de Medicina) -- Brazil

CFM Resolution 2.338/2023 requires that AI-assisted clinical decision support
tools clearly indicate that output is machine-generated and does not substitute
the physician's clinical judgment. The physician retains full responsibility for
any clinical decision.

### FDA -- United States

The FDA's guidance on Clinical Decision Support Software (2022) distinguishes
between CDS that requires a 510(k) and CDS that is exempt. Even exempt CDS must
clearly state it is intended to support, not replace, clinical judgment.

### MDR -- European Union

Under the EU Medical Device Regulation (2017/745), software qualifying as a
medical device must include instructions for use that state the intended purpose
and limitations.

## Disclaimer Templates

### Standard Clinical Disclaimer (English)

```
This response was generated by an artificial intelligence system and is intended
for informational purposes only. It does not constitute medical advice, diagnosis,
or treatment. All clinical decisions must be made by a qualified healthcare
professional based on their own assessment of the patient. The AI system's output
should be verified against current clinical guidelines and the patient's individual
circumstances.
```

### Standard Clinical Disclaimer (Portuguese)

```
Esta resposta foi gerada por um sistema de inteligencia artificial e tem carater
exclusivamente informativo. Nao constitui aconselhamento medico, diagnostico ou
tratamento. Todas as decisoes clinicas devem ser tomadas por um profissional de
saude qualificado com base em sua propria avaliacao do paciente. A saida do
sistema de IA deve ser verificada em relacao as diretrizes clinicas atuais e as
circunstancias individuais do paciente.
```

## When to Show Disclaimers

The correct strategy depends on the deployment context:

| Context | Strategy | Rationale |
|---------|----------|-----------|
| Clinical decision support | Every response | Regulatory requirement. No exceptions. |
| Patient-facing chatbot | Every response | Patients may act on AI output without consulting a provider. |
| Research assistant | First response per session + on-demand | Researchers understand limitations; per-response noise reduces usability. |
| Internal documentation lookup | First response per session | Low clinical risk; teams are trained. |

For clinical decision support, always inject the disclaimer into every response.
A "show once per session" approach is insufficient because individual responses may
be exported, printed, or shared out of context.

## TypeScript Disclaimer Injector

```typescript
interface DisclaimerConfig {
  language: "en" | "pt";
  strategy: "every_response" | "once_per_session";
  position: "prepend" | "append";
  format: "plain" | "markdown" | "html";
}

const DISCLAIMERS: Record<string, Record<string, string>> = {
  en: {
    plain:
      "DISCLAIMER: This response was generated by an AI system and is for " +
      "informational purposes only. It does not constitute medical advice. " +
      "All clinical decisions must be made by a qualified healthcare professional.",
    markdown:
      "\n\n---\n\n> **Disclaimer:** This response was generated by an AI system " +
      "and is for informational purposes only. It does not constitute medical " +
      "advice. All clinical decisions must be made by a qualified healthcare " +
      "professional.\n",
    html:
      '<div class="ai-disclaimer" role="alert">' +
      "<strong>Disclaimer:</strong> This response was generated by an AI system " +
      "and is for informational purposes only. It does not constitute medical " +
      "advice. All clinical decisions must be made by a qualified healthcare " +
      "professional.</div>",
  },
  pt: {
    plain:
      "AVISO: Esta resposta foi gerada por um sistema de inteligencia artificial " +
      "e tem carater exclusivamente informativo. Nao constitui aconselhamento " +
      "medico. Todas as decisoes clinicas devem ser tomadas por um profissional " +
      "de saude qualificado.",
    markdown:
      "\n\n---\n\n> **Aviso:** Esta resposta foi gerada por um sistema de " +
      "inteligencia artificial e tem carater exclusivamente informativo. Nao " +
      "constitui aconselhamento medico. Todas as decisoes clinicas devem ser " +
      "tomadas por um profissional de saude qualificado.\n",
    html:
      '<div class="ai-disclaimer" role="alert">' +
      "<strong>Aviso:</strong> Esta resposta foi gerada por um sistema de " +
      "inteligencia artificial e tem carater exclusivamente informativo. Nao " +
      "constitui aconselhamento medico. Todas as decisoes clinicas devem ser " +
      "tomadas por um profissional de saude qualificado.</div>",
  },
};

const sessionDisclaimed = new Set<string>();

function injectDisclaimer(
  response: string,
  sessionId: string,
  config: DisclaimerConfig
): string {
  // Check session strategy
  if (config.strategy === "once_per_session") {
    if (sessionDisclaimed.has(sessionId)) {
      return response;
    }
    sessionDisclaimed.add(sessionId);
  }

  const disclaimer = DISCLAIMERS[config.language]?.[config.format];
  if (!disclaimer) {
    // Fail safe: if no disclaimer found, use English plain
    const fallback = DISCLAIMERS.en.plain;
    return `${fallback}\n\n${response}`;
  }

  if (config.position === "prepend") {
    return `${disclaimer}\n\n${response}`;
  }

  return `${response}\n\n${disclaimer}`;
}
```

## Pipeline Integration

The disclaimer injector runs as the last step before delivery -- after citation
verification, after DDI checks, after evidence warnings. It must never be skipped
by error handling or early returns.

```typescript
async function deliverResponse(
  response: string,
  sessionId: string,
  config: DisclaimerConfig
): Promise<string> {
  // All safety gates have already run.
  // Disclaimer injection is the final step.
  const finalResponse = injectDisclaimer(response, sessionId, config);

  // Verify the disclaimer is actually present (defense in depth)
  const hasDisclaimer =
    finalResponse.includes("Disclaimer") ||
    finalResponse.includes("DISCLAIMER") ||
    finalResponse.includes("Aviso") ||
    finalResponse.includes("AVISO");

  if (!hasDisclaimer) {
    throw new Error(
      "Disclaimer injection failed. Response will not be delivered. " +
      "Session: " + sessionId
    );
  }

  return finalResponse;
}
```

## Formatting Guidelines

- Use a visually distinct style (border, background color, or horizontal rule) to
  separate the disclaimer from the AI response content.
- Use `role="alert"` in HTML for screen reader accessibility.
- Never allow the disclaimer to be collapsed, hidden, or dismissed in clinical
  decision support contexts.
- Keep the text concise. Long disclaimers get ignored. One to three sentences is
  the target.
