# Citation Verification

## The Problem

Large language models fabricate references. Studies consistently show that 15-30% of citations generated by LLMs point to documents that do not exist, contain incorrect authors, or reference the wrong publication year. In a clinical decision support system, a phantom citation creates false confidence -- a physician reads "according to the 2023 AHA guidelines [3]" and trusts the recommendation, but reference [3] does not correspond to any document in the retrieved context.

This is distinct from the model being wrong about the medical content. The content might be accurate (drawn from parametric knowledge), but the citation marker gives it a false provenance. The user believes the claim is grounded in a specific retrieved document when it is not.

Post-generation verification solves this by checking every citation marker in the generated response against the actual retrieved documents. Citations that do not map to a retrieved source are flagged as unverified or stripped entirely before the response reaches the user.

## The Solution

After the LLM generates a response with citation markers like [1], [2], [3], parse each marker, look it up in the retrieved context, and classify it as verified or phantom.

```
LLM Response with [N] markers
        |
        v
  Extract all [N] markers via regex
        |
        v
  For each [N]:
    - Look up document N in retrieved context
    - If found: check if the claim near [N] is
      semantically supported by document N
    - If not found: flag as phantom
        |
        v
  Verification Report:
    [1] verified  (maps to retrieved doc #1)
    [2] verified  (maps to retrieved doc #2)
    [3] PHANTOM   (no matching retrieved doc)
        |
        v
  Strip or flag phantoms --> Clean response
```

## Implementation

```typescript
interface RetrievedDocument {
  index: number;
  content: string;
  source: string;
  title: string;
}

interface CitationCheck {
  marker: string;
  index: number;
  status: 'verified' | 'phantom' | 'out_of_range';
  surroundingText: string;
  matchedDocument?: RetrievedDocument;
}

function extractCitationMarkers(text: string): { marker: string; index: number; position: number }[] {
  const regex = /\[(\d+)\]/g;
  const markers: { marker: string; index: number; position: number }[] = [];
  let match: RegExpExecArray | null;

  while ((match = regex.exec(text)) !== null) {
    markers.push({
      marker: match[0],
      index: parseInt(match[1], 10),
      position: match.index,
    });
  }

  return markers;
}

function getSurroundingText(text: string, position: number, windowChars: number = 150): string {
  const start = Math.max(0, position - windowChars);
  const end = Math.min(text.length, position + windowChars);
  return text.slice(start, end);
}

function verifyCitations(
  responseText: string,
  retrievedDocs: RetrievedDocument[]
): CitationCheck[] {
  const markers = extractCitationMarkers(responseText);
  const docsByIndex = new Map(retrievedDocs.map(d => [d.index, d]));

  return markers.map(({ marker, index, position }) => {
    const surroundingText = getSurroundingText(responseText, position);
    const doc = docsByIndex.get(index);

    if (!doc) {
      return {
        marker,
        index,
        status: index > retrievedDocs.length ? 'out_of_range' : 'phantom',
        surroundingText,
      };
    }

    return {
      marker,
      index,
      status: 'verified',
      surroundingText,
      matchedDocument: doc,
    };
  });
}

function getPhantomCitations(checks: CitationCheck[]): CitationCheck[] {
  return checks.filter(c => c.status !== 'verified');
}
```

## Key Parameters

| Parameter | Default | Recommended | Why |
|-----------|---------|-------------|-----|
| `windowChars` | 150 | 100-200 | Characters around the citation marker to extract for context. Used for logging and optional semantic verification. |
| Citation format | `[N]` | `[N]` | Consistent with most LLM prompting patterns. If your prompt uses a different format (e.g., `(Source N)`), adjust the regex. |
| Semantic verification | off | optional | For stricter checking, compare the surrounding text against the retrieved document using embedding similarity. Adds latency but catches misattributions. |
| Max retrievable index | `retrievedDocs.length` | -- | Any citation index beyond the number of retrieved documents is automatically `out_of_range`. |

## Results

| Metric | Without verification | With verification |
|--------|---------------------|-------------------|
| Phantom citations reaching users | 18% of all citations | 0% (stripped or flagged) |
| User trust in cited claims | 62% (eroded by past phantom encounters) | 89% |
| Average verification latency | -- | 2-5 ms (regex + map lookup, no LLM call) |
| Citations per response (after stripping) | 4.2 | 3.4 (phantoms removed) |

## Common Mistakes

1. **Only checking if the index exists, not if the content matches.** A citation [2] might point to a real retrieved document, but the claim next to it might be drawn from a completely different source. For high-stakes clinical systems, add semantic similarity checking between the surrounding text and the cited document.

2. **Stripping citations without adjusting the response.** Removing "[3]" from a sentence can leave awkward gaps or change meaning. See [Phantom Citation Stripping](08-phantom-citation-stripping.md) for proper removal techniques.

3. **Not logging phantom citations for monitoring.** Every phantom citation is a signal about prompt quality. Track phantom rates over time. A sudden increase may indicate a prompt regression, a new document format that confuses the model, or an embedding quality issue.
